FAQ
Common questions.
What exactly is Starvox?▾
Starvox is a self-hosted Master AI Company OS. It's a multi-agent system where 16 specialist AI agents (PM, frontend, backend, QA, security, DevOps, and more) collaborate to take your natural language intent and produce shipped, deployed, tested software — automatically.
How is this different from GitHub Copilot or Cursor?▾
Copilot and Cursor assist individual developers in their editor. Starvox replaces the entire delivery loop — spec, code, test, deploy, monitor — without a developer in the loop. It's a company-level OS, not a code autocomplete tool.
Which AI models does Starvox use?▾
Starvox routes tasks across 25 models from 10 providers — Claude, GPT-4o, Gemini, DeepSeek, Mistral, and more. The router picks the best model for each sub-task based on cost, speed, and capability profile. You can override routing rules per project.
Is my code safe? Where does it run?▾
Starvox is self-hosted — your code never leaves your infrastructure. The build sandbox is an isolated container on your own machine. API calls go to your own provider accounts. We never store your code or API keys.
Can I use Starvox for my existing products?▾
Yes. Starvox works on any existing codebase. Point it at your repo, describe a task, and the agents read your project context before writing or deploying anything. It's designed for portfolios of products, not just greenfield builds.
